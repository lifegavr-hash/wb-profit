# План замены домена и имени проекта (find-replace)

Когда выберешь новое имя и купишь домен, делать замены в **строгом порядке**, чтобы ничего не сломалось. Я (Claude) могу сделать все замены за тебя — нужно только финальные значения для подстановки.

- **Стало:** `[НОВОЕ_НАЗВАНИЕ]`

## Порядок выполнения (когда всё будет готово)

### Этап 1. Подготовка (твоё участие)

1. Выбрать **новое имя** проекта (решаем завтра).
2. Купить **новый домен** (REG.RU/Beget — выбор провайдера в `ROADMAP.md` п.1.2).
3. Завести **корпоративный email** (`support@домен`) — обычно идёт в комплекте с доменом или через Яндекс 360.
4. Решить про **Telegram-бот** — оставляем или вычищаем.

### Этап 2. Замены в коде (моё участие, я делаю)

Я могу сделать все текстовые замены за один заход:

```bash
# Бренд
find wb-profit/public/ -type f \( -name "*.html" -o -name "*.svg" \) \
  -exec sed -i '' 's/WB Profit/[НОВОЕ_НАЗВАНИЕ]/g' {} \;

# Домен
sed -i '' 's|wb-profit\.vercel\.app|[новый-домен].ru|g' wb-profit/public/*.html

# Email
sed -i '' 's/ooovsktrans@mail\.ru/support@[новый-домен].ru/g' wb-profit/public/*.html
```

После замен — проверим, что нигде не сломалось (grep'ом найдём остатки).

### Этап 3. Привязка домена к Vercel (твоё участие)

1. В Vercel Dashboard → Project → Settings → **Domains** → Add Domain.
2. Указать новый домен.
3. Vercel даст DNS-записи (обычно CNAME или A-record) — добавить их у регистратора.
4. Подождать 5-30 минут до распространения DNS.
5. Vercel автоматически выдаст SSL-сертификат.

### Этап 4. Обновление Supabase (моё + твоё участие)

1. В Supabase Dashboard → Authentication → URL Configuration → обновить Site URL и Redirect URLs на новый домен.
2. Email-шаблоны (которые ты вставишь по `EMAIL_TEMPLATES.md`) — все ссылки на новый домен.

### Этап 5. Документы и SEO

1. Обновить `og-image.svg` — нарисовать новый.
2. Обновить `favicon.svg` — новая буква.
3. В `sitemap.xml` и `robots.txt` поправить URL'ы (я сделаю).
4. Подать новый сайт в Яндекс.Вебмастер и Google Search Console.
5. Настроить **301-редиректы** со старого `wb-profit.vercel.app` на новый домен (через Vercel rewrites в `vercel.json`) — чтобы старые ссылки не отдавали 404.

### Этап 6. Внешние интеграции

1. Подключённая Яндекс.Метрика — поменять домен в настройках счётчика.
2. JSON-LD на лендинге — обновить URLs и название организации.
3. Если успел опубликовать в каталогах/обзорах — попросить обновить ссылки.

## Что я НЕ могу заменить автоматически

1. **og-image.svg** — там графика, нужен дизайнер или Figma.
2. **favicon.svg** — переделать букву.
3. **Документы оферты и политики** — там есть юридические формулировки про название юр.лица/ИП. Если юр.лицо меняется — нужна юр.проверка. Если только название продукта — текст оферты/политики надо переписать в местах «Сервис называется "WB Profit"» (это твоя задача либо с юристом).
4. **Содержание `EMAIL_TEMPLATES.md`** — все шаблоны писем содержат «WB Profit» в теле, теме, подвалах. Я переделаю на новое имя за пару минут — это просто файл с текстами.
