# Уведомление в Роскомнадзор как оператор ПД

**Что это:** черновик данных для электронного уведомления о намерении осуществлять обработку персональных данных. Подаётся через сайт РКН.

**Ссылка для подачи:** https://pd.rkn.gov.ru/operators-registry/notification/form/

**Стоимость:** бесплатно. **Срок рассмотрения:** до 30 дней.

---

## ⚠️ Что нужно подготовить до подачи

1. **Реквизиты ИП** (если ИП ещё не зарегистрировано — это блокер).
2. **Юридический адрес** регистрации ИП.
3. **СНИЛС** заявителя.
4. **Электронная подпись** для подачи через Госуслуги (необязательно, можно подать без ЭП — тогда нужно отправить заверенную копию почтой).
5. **Решение** про название проекта (если ребрендим до подачи — указываем новое название; если после — будет уточнение в реестре).

---

## Заполнение формы по разделам

### 1. Сведения об операторе

| Поле | Значение |
|---|---|
| Полное наименование | ИП [Фамилия Имя Отчество] |
| ОГРНИП | [номер из выписки] |
| ИНН | [12 цифр] |
| Адрес регистрации | [полный адрес из паспорта/выписки] |
| Адрес для корреспонденции | (тот же) |
| Контактный телефон | [твой номер] |
| Email | support@[новый-домен].ru *(после смены домена; пока — текущий)* |
| Сайт | https://[новый-домен].ru *(после смены домена; пока swprofit.ru)* |

### 2. Дата начала обработки ПД

> Указать **фактическую дату** запуска сервиса в прод (например, 01.01.2024).

### 3. Правовое основание обработки ПД

> Согласие субъекта персональных данных (статья 9 152-ФЗ), договор-оферта между Оператором и Пользователем (статья 6 ч.1 п.5 152-ФЗ).

### 4. Цель обработки ПД

> Оказание услуг по предоставлению доступа к веб-сервису аналитики продаж селлеров маркетплейсов (Wildberries и другие), включая регистрацию пользователей, обработку платежей за подписку, направление сервисных и информационных уведомлений, поддержку пользователей.

### 5. Категории ПД, которые обрабатываются

Выбрать (галочки):
- Фамилия, имя, отчество
- Адрес электронной почты
- Номер телефона (если собираешь)
- Данные платёжных операций (через платёжного агрегатора — указать ЮKassa после подключения)

**НЕ ВЫБИРАТЬ:**
- Биометрические ПД
- Специальные категории ПД (раса, здоровье, политика, религия)

### 6. Категории субъектов

- Физические лица — пользователи сервиса (включая индивидуальных предпринимателей в личном качестве).

### 7. Перечень действий с ПД

Выбрать:
- Сбор
- Запись
- Систематизация
- Накопление
- Хранение
- Уточнение (обновление, изменение)
- Извлечение
- Использование
- Удаление
- Уничтожение

### 8. Способы обработки

- Автоматизированная обработка (через программные средства).

### 9. Меры по обеспечению безопасности

> Применяются организационные и технические меры: хеширование паролей (bcrypt), HTTPS-шифрование передачи данных (TLS 1.3), Row Level Security в БД (Supabase Postgres), ограничение доступа сотрудников Оператора к ПД пользователей служебной необходимостью, регулярное резервное копирование, журналирование событий безопасности.

### 10. Трансграничная передача ПД

⚠️ **Это важный пункт.** Текущая БД — Supabase AWS Frankfurt.

**Варианты:**

**A. Честный (рекомендую):**
> Осуществляется в Германию (Федеративная Республика Германия) — на серверы AWS Frankfurt (хостинг провайдера Supabase Inc.). Германия включена в перечень стран, обеспечивающих адекватную защиту прав субъектов ПД (приказ Роскомнадзора № 274 от 05.08.2022, ред. от 19.10.2023).

**B. Если планируешь миграцию в РФ в ближайшее время:**
> Не осуществляется. Данные хранятся на серверах в Российской Федерации (Yandex Cloud / VK Cloud / Selectel — выбрать).

### 11. Местонахождение БД с ПД граждан РФ

⚠️ **Самый чувствительный пункт.** По 152-ФЗ ст.18 — должна быть в РФ.

- Если БД в Supabase (AWS Frankfurt) — указать честно «за пределами РФ» и в графе «срок устранения нарушения» указать планируемую дату миграции.
- Если уже мигрировал — указать страну РФ и название хостинг-провайдера.

---

## После подачи

1. РКН рассматривает до 30 дней.
2. Получаешь номер в реестре операторов ПД.
3. На сайте `pd.rkn.gov.ru` появляется твоя запись (можно проверить публично — это плюс к доверию пользователей).
4. В `privacy.html` нужно будет обновить блок «регистрация в процессе» на «зарегистрирован в реестре операторов ПД под номером XX-XX-XXXXXX».

---

## Чего НЕ делать

- Не подавай уведомление от физлица — нужен статус ИП или юр.лица.
- Не врать про местонахождение БД — РКН проверяет.
- Не игнорируй пункт о цели — слишком общая формулировка («любая деятельность») вызовет вопросы.
